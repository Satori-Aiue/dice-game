# 骰子赌博游戏

多人 AI 骰子赌博模拟游戏。4 名 AI 玩家各持随机风险偏好，通过三轮下注 + 概率计算进行理性决策，直到有人破产。

## 快速开始

```bash
pip install pytest

python main.py                    # 默认 4 人，每人 $100
python main.py --seed 42          # 固定随机种子（复现同一局）
python main.py --quiet --seed 42  # 静默模式（只看最终排名）
python main.py --debug --seed 42  # 调试模式（显示每个决策的 EV 计算过程）
python main.py --players 4 --money 200 --seed 42
python main.py --names Alice Bob Charlie Diana

python -m pytest tests/ -v                          # 全部测试
python -m pytest tests/test_judge.py -v             # 单个测试文件
python -m pytest tests/test_judge.py::TestJudge::test_exact_13_wins -v  # 单个测试
```

## 游戏设定

| 项目 | 设定 |
|---|---|
| 玩家数量 | 4 人 |
| 初始资金 | 每人 $100 |
| 骰子面值 | 2 ~ 7（均匀分布，无作弊） |
| 目标点数 | **13 点** |
| 最低下注 | $5 |
| 最低加注 | $6（5 + 1），金额为整数 |
| 下注上限 | 不超过个人全部资金 |
| 结束条件 | 一轮结束时有人资金 < $5（视为失去全部资金，游戏结束） |

### All-in 规则

资金最少的玩家如果加入了自己的全部资金，则该资金作为赌注的最大值，其余人下注必须等于该值（或者退出）。

### 等额原则

下注必须保证所有人下注的资金均相同后才能进行下一步行动，否则依次循环加注。

## 游戏流程

```
发牌 → 第一轮下注 → 选择公共骰子 + 第二轮下注 → 公开骰子 + 投公共骰子 → 第三轮下注 → 结算
```

### 阶段 1：发牌

每人投掷 2 颗暗骰（点数仅自己可见）。结算时总点数为两枚（或三枚）骰子的和。

### 阶段 2：第一轮下注（强制）

每人**固定下注 $5**，不可弃牌，不可加注。下注金额受 All-in 规则约束（不得超过最小资金玩家的总资金）。

### 阶段 3：选择公共骰子 + 第二轮下注

**选择公共骰子**：每人秘密决定是否加入第三颗公共骰子。选择结果不公开。当前策略：2 骰和 ≤ 9 则加入，> 9 则不加入。

**第二轮下注**：可弃牌（失去第一轮注额），可跟注/加注。每次加注至少 $6，金额为整数。公共骰子尚未投掷，玩家基于期望值决策。

循环加注直到所有人下注额相等，遵循 All-in 规则。

### 阶段 4：公开骰子 + 投公共骰子

每人随机选择自己的一颗暗骰公开点数（所有人可见）。系统投出公共骰子，所有选择加入的玩家共享此点数。

### 阶段 5：第三轮下注（最终轮）

可弃牌（失去前两轮注额），可跟注/加注。此时已知自己的最终点数、所有人的公开骰、公共骰点数，可以精确推断对手得分分布。

### 阶段 6：结算（拼点）

所有存活玩家亮牌比点。

---

## 结算规则

所有骰子都是 2~7，不存在作弊行为。结算时总点数为三枚（或两枚骰子）的和。

**规则 1（平局优先权）**：选择公共骰子的玩家在平局时单独获胜，点数相同记平局，适用于后面所有规则。

**规则 2（恰好 13 点）**：点数等于 13 则获胜。

**规则 3（均小于 13）**：均小于 13 则点数最大的玩家获胜。

**规则 4（爆牌处理）**：大于 13 则出局（如果场上还有小于等于 13 的玩家）。但如果所有人点数都大于 13，则点数最小的玩家获胜。

**规则 5（奖池分配）**：获胜的玩家获得全部奖金，多人平局则平分全部奖金。

## AI 策略系统

每个 AI 玩家在游戏开始时被随机分配一个**风险厌恶度 α** ∈ [0.05, 0.55]，一局游戏内不变。不同 α 模拟不同策略风格：

| α 范围 | 风格 | 行为特征 |
|---|---|---|
| 0.05 ~ 0.10 | 激进 | 几乎无视爆牌风险，倾向留在牌局 |
| 0.10 ~ 0.25 | 偏激进 | 较愿意冒险 |
| 0.25 ~ 0.40 | 中性 | 平衡 |
| 0.40 ~ 0.50 | 偏保守 | 较谨慎 |
| 0.50 ~ 0.55 | 保守 | 对爆牌极度敏感，倾向弃牌 |

### 第二轮策略

**文件**：`ai/betting_strategy_round2.py`

**核心思路**：第二轮信息不足以评估对局胜率（不知道对手手牌、对手是否加入公共骰、公共骰结果），不做对手建模，仅做自身牌力评估。

**四步框架**：

1. **构建概率分布** — `build_score_distribution(hand, has_public)`：枚举第三骰 2~7（各 1/6），得到最终得分分布
2. **提取统计量** — P(13)、P(安全区)、P(爆牌)、安全区期望分数 E_safe、爆牌区期望分数 E_bust
3. **计算 HandStrength** — `compute_hand_strength(hand, has_public, risk_aversion)`：
   ```
   value_13   = P(13) × tie_adv        （选公共骰有 5% 平局优势加成）
   value_safe = P(安全) × (E_safe/13)³  （非线性，离 13 越近越高）
   base       = value_13 + (1 − value_13) × value_safe
   strength   = base − α × P(爆牌)      （风险惩罚）
   ```
4. **决策** — `decide_second_bet()`：
   - strength < 0.35 → 弃牌
   - 0.35 ≤ strength < 0.75 → 跟注
   - strength ≥ 0.75 → 加注至目标注额（目标注额 = 可支配资金 × strength 映射 × 20%）

**α 的作用**：`bust_penalty = α × P(爆牌)`。同一手牌（如 P(爆牌)=50%），激进型（α=0.06）牌力 ≈ 0.36 选择跟注，保守型（α=0.55）牌力 ≈ 0.12 选择弃牌。

### 第三轮策略

**文件**：`ai/betting_strategy_round3.py`

**核心思路**：第三轮信息最充分——自己的最终点数已确定，可以利用公开骰做精确的对手建模。

**四步框架**：

1. **贝叶斯推断** — `build_opponent_distribution(revealed_die, public_die)`：由对手公开骰 + 第二轮规则（≤9 加入公共骰）反推每位对手的 (score, chose_public) 概率分布。使用软规则（95% 遵循率，允许 5% 偏离）。

2. **Monte Carlo 桌面胜率** — `monte_carlo_win_probability()`：500 次采样，从每位对手的分布中抽样，模拟 showdown（`judge_one_vs_one`），统计真实的 P(击败所有对手)。

3. **风险调整期望效用** — `risk_adjusted_eu()`：
   ```
   EU = P(win) × (pot + call) − (1 − P(win)) × call × (1 + α)
   ```
   α 作用于效用函数的**损失项**，不修改胜率本身。保守型玩家（高 α）的损失被放大，需要更高胜率才愿意跟注。

4. **决策** — `decide_third_bet()`：
   - P(win) < 2% → 弃牌（绝望手）
   - EU < 0 → 弃牌
   - P(win) ≥ 55% → 加注（步长 3×MIN_RAISE = $18，确保快速收敛）
   - 其他 → 跟注
